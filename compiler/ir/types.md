# IR Types

All IR operands, locals, globals, function signatures, and instruction results
carry an `IrType`.

The implementation source of truth is `src/ir/types.ts`.

```ts
type IrType =
  | IrPrimitiveType
  | IrNamedType
  | IrNullableType
  | IrTupleType
  | IrFunctionValueType
  | IrUnknownType
  | IrErrorType;
```

## Primitive Types

Primitive types are represented by a single tagged interface.

```ts
interface IrPrimitiveType {
  kind: "primitive";
  name:
    | "i8" | "i16" | "i32" | "i64"
    | "u8" | "u16" | "u32" | "u64"
    | "f32" | "f64"
    | "bool"
    | "string"
    | "void"
    | "null";
}
```

Rules:

- `void` appears as the return type for functions that do not return a value.
- `null` is the type of the null literal before it is wrapped into a nullable
  representation.
- `string` is heap-backed by the runtime.

## Named Types

```ts
interface IrNamedType {
  kind: "named";
  name: string;
  args: IrType[];
  decl?: "struct" | "enum";
}
```

`IrNamedType` represents source-level named types after checking and
specialization.

Rules:

- Struct and enum values use `decl: "struct"` or `decl: "enum"` when the type
  refers to an emitted aggregate declaration.
- Runtime arrays use `name: "Array"` with one element type argument.
- Generic named types must be specialized before they reach emittable IR.

## Composite Types

```ts
interface IrNullableType {
  kind: "nullable";
  base: IrType;
}

interface IrTupleType {
  kind: "tuple";
  elements: IrType[];
}
```

Rules:

- Nullable values lower through explicit nullable instructions.
- Tuple values are structural and are emitted through canonical C tuple types.
- Tuple and nullable C helper declarations are generated by the emitter from
  type use; they are not separate IR declarations.

## Function Value Types

Function declarations use `IrFunctionType`; function values use the same shape
inside `IrFunctionValueType`.

```ts
interface IrFunctionType {
  params: IrParamType[];
  returnType: IrType;
}

interface IrFunctionValueType {
  kind: "function";
  params: IrParamType[];
  returnType: IrType;
}

interface IrParamType {
  type: IrType;
  mutable: boolean;
}
```

Rules:

- Parameter names are not part of the function type.
- `mutable` records the source-level parameter mutability that still matters to
  lowering and emission.
- Function and method return types may be inferred by the checker. The lowerer
  must use the checker-recorded function type when no explicit `-> T`
  annotation is present.

## Unknown and Error Types

```ts
interface IrUnknownType {
  kind: "unknown";
}

interface IrErrorType {
  kind: "error";
}
```

These are recovery sentinels. They should not appear in successfully emittable
IR.
